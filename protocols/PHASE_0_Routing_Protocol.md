# 🚀 The Phase 0 Routing Protocol

This is our universal starting point for every new project. **Phase 0** acts as the launchpad where we define the vision and then *route* the project into the correct execution framework (F.O.R.G.E. or B.L.A.S.T.).

## Step 1: The Concept Pitch & Routing
* **USER:** "Let's start a new project..." + (The initial concept)
* **ASSISTANT:** Evaluates the concept and routes it to the correct framework:
  * **Option A: F.O.R.G.E. (Web Apps & UIs)** → The project has a user interface, visual elements, or is a frontend-heavy application.
  * **Option B: B.L.A.S.T. (Automations & Agents)** → The project is a backend automation script, data pipeline, AI agent, or "invisible" system.

## Step 2: The Framework-Specific Discovery
Once routed, the Assistant initiates the Discovery Phase of the chosen framework.

### If F.O.R.G.E. (Web Apps)
We answer the 5 Discovery Questions:
1. **The North Star:** Primary purpose of the app.
2. **The Vibe:** Visual aesthetic (e.g., dark mode, glassmorphism).
3. **Core Interactions:** What the user clicks/drags.
4. **Data Concept:** Where the data comes from (plain English).
5. **Dealbreakers:** Strict constraints.

### If B.L.A.S.T. (Automation)
We answer the 5 Discovery Questions:
1. **The North Star:** Singular desired outcome.
2. **Integrations:** External services needed (e.g., Slack, Stripe).
3. **Source of Truth:** Where the primary data lives.
4. **Delivery Payload:** How the final result is delivered.
5. **Behavioral Rules:** System rules (Tone, "Do Nots").

## Step 3: Tech Stack Proposal
The Assistant proposes the exact technical stack for the project in **plain English**, explaining why it is the best tool for the job. The User approves without needing technical knowledge.

## Step 4: Blueprint Generation (Project Memory)
The Assistant generates the foundational documents (no code yet):
* `task_plan.md` - Phase-by-phase goals.
* `architecture.md` - Flow and layers.
* `schema.md` - Data shapes.

## Step 5: The Mental Walkthrough
We verbally trace the user journey or data flow from start to finish to catch logic holes.

## Step 6: The Green Light & Epic Planning
The User approves the Blueprint. We break the work into large chunks ("Epics"). The User can deploy the Assistant to execute an Epic (e.g., overnight execution).
