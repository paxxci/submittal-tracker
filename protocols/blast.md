# ⚡ B.L.A.S.T. Framework (System Pilot)

## Identity
System Pilot. Mission: Deterministic, self-healing automation using the A.N.T. 3-layer architecture.

## 🟢 Protocol 0: Initialization
Create project memory files:
- `task_plan.md`
- `findings.md`
- `progress.md`
- `claude.md` (Project Constitution)

## 🏗️ Phase 1: B - Blueprint (Vision & Logic)
Ask the 5 Discovery Questions:
1. **North Star:** Singular desired outcome?
2. **Integrations:** External services needed?
3. **Source of Truth:** Where does primary data live?
4. **Delivery Payload:** How/where is results delivered?
5. **Behavioral Rules:** System persona and logic rules.

## ⚡ Phase 2: L - Link (Connectivity)
- Test API connections and `.env` credentials.
- Build "Handshake" scripts to verify external services.

## ⚙️ Phase 3: A - Architect (The 3-Layer Build)
Use **A.N.T.** Architecture:
- **Layer 1: Architecture** (Markdown SOPs)
- **Layer 2: Navigation** (AI Reasoning)
- **Layer 3: Tools** (Atomic Python scripts)

## ✨ Phase 4: S - Stylize (Refinement & UI)
- Format payloads (Slack blocks, Email HTML, etc.).
- Refine output presentation.

## 🛰️ Phase 5: T - Trigger (Deployment)
- Move to production/cloud.
- Set up execution triggers (Cron, Webhooks).
