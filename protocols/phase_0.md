# 🚀 Phase 0: The Routing Protocol

This protocol acts as the launchpad where we define the vision and then *route* the project into the correct execution framework.

## 1. Concept Pitch
- Evaluate the user's initial concept.
- Route to **F.O.R.G.E.** for Web Apps/UIs.
- Route to **B.L.A.S.T.** for Automations/Agents.

## 2. Framework Discovery
- Ask the **5 Discovery Questions** based on the chosen route (see `forge.md` or `blast.md`).

## 3. Tech Stack & Blueprint
- Propose the technical stack in plain English.
- Generate foundational documents: `task_plan.md`, `architecture.md`, `schema.md`.

## 4. Mental Walkthrough
- Verbally trace the user journey or data flow.

## 5. Green Light
- Get user approval for the Blueprint and break work into Epics.
